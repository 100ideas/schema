ContactPoint

Status: unstable, Role: tertiary

A contact point, for example, a R&D department.

Properties

type (required) enum<​ContactPoint​> The name of the type and all descendant types. Entity
alternateNames array<​string​> Alternate names (aliases) for the item. Thing
availableLanguages array<​string​> Languages (human not programming) in which it is possible to communicate with the organization/department etc. ContactPoint
description string | array<​Node​> A description of the item. Thing
emails array<​string<email>​> Email address for correspondence. ContactPoint
id string The identifier for this item. Entity

url string<uri> The URL of the item. Thing

The ContactPoint type allows you to provide details about an contact information such as languages (human) available for communication, emails and telephone numbers. This type of often used to describe the ContactPoints of an Organization.

Examples

A contact point with email, telephone and two supported languages.

{
  "type": "ContactPoint",
  "availableLanguages": ["English", "Māori"],
  "emails": ["office@otago.ac.nz"],
  "telephone": "00641234567"
}

JATS

ContactPoint is analogous, and structurally similar to, the JATS <corresp> element.

A journal author <corresp> links to a number of JATS elements which are also properties of ContactPoint in Stencila schema. These JATS elements are:

emails email

HTML5

ContactPoint is analogous to the HTML5 <address> element. The <address> element represents the contact information for the section it applies to.

  itemtype="https://schema.stenci.la/Entity">{type:'ContactPoint',availableLanguages:['English','Māori'],emails:['office@otago.ac.nz'],telephoneNumbers:['00641234567']}</span>
