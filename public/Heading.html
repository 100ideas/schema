Heading

Properties

content (required) array<​InlineContent​> Content of the heading. Heading
depth (required) number The depth of the heading. Heading
type (required) enum<​Heading​> The name of the type and all descendant types. Entity
id string The identifier for this item. Entity
meta object Metadata associated with this item. Entity

Examples

To illustrate how Heading nodes are represented in alternative formats, we'll use the following example.

{
  "type": "Heading",
  "depth": 2,
  "content": ["Secondary Heading"]
}

For compatibility with HTML, only integer depths in the range 1–6 are supported.

Encodings

This section describes common encodings for this node type. These samples are generated from the above examples by Encoda, but you can also author them in each format.

HTML

HTML supports Heading nodes with the <h1> to <h6> elements.

<h2 id="secondary-heading">Secondary Heading</h2>

YAML

Markdown

Markdown headings are denoted by a number of hashes (#) equal to the depth. 

## Secondary Heading
