A Table type allows you to represent two-dimensional data, in a way that can easily be translated into an XHTML table if needed. It is primarily a container for Table Row and Table Cell types.

Examples

Simple

This is the most basic form of Table you can have. It contains no rows or columns, and represents an empty table.

{
  "type": "Table",
  "rows": []
}

Table with Rows & Cells

The Table type can contain rows, which in turn contain cells.

{
  "type": "Table",
  "rows": [
    {
      "type": "TableRow",
      "cells": [
        {
          "type": "TableCell",
          "content": ["one"]
        },
        {
          "type": "TableCell",
          "content": ["two"]
        }
      ]
    },
    {
      "type": "TableRow",
      "cells": [
        {
          "type": "TableCell",
          "content": ["three"]
        },
        {
          "type": "TableCell",
          "content": ["four"]
        }
      ]
    }
  ]
}

JATS

Table is analogous, and structurally similar to, the JATS <table> type. They both store data in a way that is "intended to be converted easily to the XHMTL table element."

mdast

Table is analogous to the mdast Table node type, however note that unlike the mdast equivalent, the Stencila variant does not contain the stylistic align field.

OpenDocument

Table is analogous to the OpenDocument <table:table> element, which is the root element for a table.
